# Research Findings

## Part 1: Codebase Analysis

### Project Structure
- **File-based routing** with TanStack Router: routes in `src/routes/`, auto-generated route tree
- **Feature-based architecture**: `src/features/{ontology,compliance,reports}/` each with api, types, components, context
- **4 existing routes**: `/` (home), `/ontology`, `/compliance`, `/reports`

### Routing Pattern
- `createFileRoute()` with component and optional `validateSearch` for type-safe search params
- Ontology page uses search validation: `{ view?: "graph" | "tree" | "compare"; concept?: string; frameworks?: string }`
- Route tree auto-generated by TanStack Router Vite plugin

### API Hooks (TanStack Query)
- Query key factory: `ontologyKeys.all`, `.frameworks()`, `.concepts(fwId)`, `.relationships()`, `.search(q, fwId)`
- Available hooks: `useFrameworks()`, `useConcepts(fwId)`, `useConceptRelationships(id)`, `useRelationships()`, `useSearchConcepts(q, fwId)`, `useTopics()`
- Stale times: Frameworks=Infinity, Concepts=5min, Search=30s
- Paginated responses: `PaginatedResponse<T>` with data, total, page, limit, total_pages

### Types
- `Framework`: id, name, version, description, source_url
- `Concept`: id, framework_id, parent_id, concept_type, code, name_en/nb, definition_en/nb, source_reference
- `Relationship`: id, source_concept_id, target_concept_id, relationship_type, description
- `ConceptWithRelationships`: Concept + related_concepts[]
- `GraphNode/GraphEdge/GraphData`: D3-compatible graph types
- `TreeNode/FrameworkTreeData`: Hierarchical tree types
- `ExplorerState`: UI state (selectedConceptId, viewMode, activeFrameworks, activeTopics, etc.)

### Design System
- **Fonts**: IBM Plex Mono (headings, code) + IBM Plex Sans (body)
- **Colors**: oklch color space, deep slate primary with amber accents
- **21 framework colors** in `graphTransform.ts` via `getFrameworkColor(id)`
- **CSS classes**: `.topo-grid`, `.gradient-mesh`, `.feature-card`, `.tech-badge`, `.stat-number`, `.corner-markers`, `.connection-line`
- **Animations**: fadeInUp, slideInLeft, drawLine, pulseGlow with `.delay-100` through `.delay-600`
- **Dark mode**: Full oklch dark theme override

### UI Components (shadcn/ui)
- Available: Button, Card, Dialog, Input, Label, Select, Badge
- Pattern: CVA variants, cn() utility, Radix UI primitives
- Icon library: lucide-react

### Dependencies
- React 19.2, TanStack Router 1.158, TanStack Query 5.90, D3 7.9, Axios 1.13
- Vitest 4.0 (available but no test files exist), TypeScript 5.9, Tailwind CSS 4.1
- html2canvas 1.4 for export

### Patterns for New Pages
1. Route: `createFileRoute()` + component + optional validateSearch
2. Data: TanStack Query hooks with proper stale times
3. Layout: Three states (loading skeleton, error, data) + animation classes
4. Styling: Combine Tailwind utilities with design system CSS classes
5. Components: Break into smaller pieces (List, Card, Filters)

## Part 2: Web Research

### Topic 1: React Heatmap/Matrix Visualization

**Recommended: Custom SVG approach (D3 for math, React for rendering)**
- 22x22 = 484 cells â€” well within SVG performance limits (Canvas overkill)
- D3 `scaleBand()` for axes, `scaleSequential()` for color encoding
- React renders `<rect>` elements, D3 handles scale math in useMemo
- Alternative: Nivo `@nivo/heatmap` if we want a declarative API

**Performance**: Wrap scale calculations in useMemo, use `scaleSequentialSqrt` to reduce extreme value dominance

**Accessibility**:
- Never rely on color alone â€” add data labels or symbols
- Minimum 3:1 contrast between adjacent colors
- Discrete color bins > continuous ramps for colorblind users
- Keyboard navigation with Arrow keys between cells
- Provide tabular data alternative
- Respect prefers-reduced-motion

### Topic 2: TanStack Router File-Based Routing

**File naming conventions**:
- `routes/frameworks/index.tsx` â†’ `/frameworks`
- `routes/frameworks/$id.tsx` â†’ `/frameworks/:id` (dynamic segment)
- `routes/crosswalk/index.tsx` â†’ `/crosswalk`
- `routes/concepts/search.tsx` â†’ `/concepts/search`

**Search params for filter state**:
```typescript
validateSearch: (search) => ({
  q: (search.q as string) || '',
  frameworks: ((search.frameworks as string) || '').split(',').filter(Boolean),
  type: (search.type as string) || '',
})
```

**Key rules**: $param for dynamic segments, dot for flat nesting, - prefix to exclude files

### Topic 3: Faceted Search UI Patterns

**URL State**: Use TanStack Router's validateSearch â€” natural fit since project already uses it

**Debounce**: 300ms for text input, update URL params after debounce settles, TanStack Query keyed to search params

**Filter Sidebar**:
- Checkbox lists with counts per option
- Collapsible sections for each facet (framework, concept type, relationship type)
- Active filters shown as removable pills above results
- Order facet values by count (descending)

**Keyboard Navigation**:
- Auto-focus search on page load
- Arrow keys for results, Enter to select
- Tab between filter groups, Space to toggle
- Escape to clear search

**Component Structure**: SearchInput â†’ ActiveFilters â†’ FilterSidebar â†’ ResultsList
