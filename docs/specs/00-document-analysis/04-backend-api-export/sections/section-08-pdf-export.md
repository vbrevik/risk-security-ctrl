# Section 08: PDF Export

## Overview

This section implements the `export_pdf.rs` module inside `backend/src/features/analysis/`. It provides a single public function `generate_pdf` that produces an in-memory PDF document containing: a title page, executive summary, coverage heatmap chart, per-framework sections with radar charts and findings tables, a priority breakdown chart, and an appendix with truncated source text. Charts are generated by calling functions from the `charts.rs` module (section 07).

**Dependency:** Section 07 (Chart Rendering) must be completed first. The chart functions `render_coverage_heatmap`, `render_radar_chart`, and `render_priority_chart` are called to produce PNG bytes that are embedded as images in the PDF.

## Key Types Used

From `backend/src/features/analysis/models.rs` (already exists):

- `Analysis` -- the full analysis record with fields like `name`, `description`, `status`, `matched_framework_ids`, `processing_time_ms`, `token_count`, `created_at`
- `AnalysisFindingWithConcept` -- finding data JOINed with concept metadata (`concept_code`, `concept_name_en`, `concept_definition_en`, `finding_type`, `confidence_score`, `priority`, `evidence_text`, `recommendation`, `framework_id`)

The `frameworks` parameter is `&[(String, String)]` -- a slice of `(framework_id, framework_display_name)` tuples, assembled by the export handler (section 10) before calling this function.

## Crate Dependency

The `genpdf` crate with the `images` feature must be added to `backend/Cargo.toml` (handled in section 01):

```toml
genpdf = { version = "0.2", features = ["images"] }
```

The `image` crate (version 0.25) is also required for decoding chart PNGs before embedding.

## Font Dependency

LiberationSans TTF fonts must be present at `backend/fonts/`:

- `LiberationSans-Regular.ttf`
- `LiberationSans-Bold.ttf`
- `LiberationSans-Italic.ttf`

These are Apache-licensed and downloaded/committed in section 01. The `generate_pdf` function loads them via `genpdf::fonts::from_files("./fonts", "LiberationSans", ...)`. If fonts are missing, the function returns `ExportError` rather than panicking.

## Tests

Tests go in `backend/src/features/analysis/export_pdf.rs` as a `#[cfg(test)] mod tests` block.

### test_generate_pdf_returns_bytes

Call `generate_pdf` with a fabricated `Analysis` (status completed, some matched_framework_ids) and a small vec of `AnalysisFindingWithConcept` entries plus a frameworks list. Assert the result is `Ok`, the returned `Vec<u8>` is non-empty, and the first 4 bytes are `%PDF` (bytes `0x25, 0x50, 0x44, 0x46`).

```rust
#[test]
fn test_generate_pdf_returns_bytes() {
    let analysis = make_test_analysis();
    let findings = make_test_findings();
    let frameworks = vec![("fw-1".into(), "Test Framework".into())];
    let result = generate_pdf(&analysis, &findings, &frameworks);
    assert!(result.is_ok());
    let bytes = result.unwrap();
    assert!(!bytes.is_empty());
    assert_eq!(&bytes[0..4], b"%PDF");
}
```

### test_generate_pdf_contains_analysis_name

Render a PDF with an analysis whose name is a distinctive string (e.g., `"UniqueReportTitle2026"`). Convert the result bytes to a lossy UTF-8 string and assert it contains the analysis name. This verifies the title page content is present.

```rust
#[test]
fn test_generate_pdf_contains_analysis_name() {
    let mut analysis = make_test_analysis();
    analysis.name = "UniqueReportTitle2026".to_string();
    let findings = make_test_findings();
    let frameworks = vec![];
    let result = generate_pdf(&analysis, &findings, &frameworks).unwrap();
    let text = String::from_utf8_lossy(&result);
    assert!(text.contains("UniqueReportTitle2026"));
}
```

### Test helper stubs

Both tests need helper functions `make_test_analysis() -> Analysis` and `make_test_findings() -> Vec<AnalysisFindingWithConcept>` that return minimal valid instances. These are local to the test module.

## Implementation Details

### File: `backend/src/features/analysis/export_pdf.rs`

#### Public function signature

```rust
pub fn generate_pdf(
    analysis: &Analysis,
    findings: &[AnalysisFindingWithConcept],
    frameworks: &[(String, String)],
) -> Result<Vec<u8>, ExportError>
```

#### Document structure (sequential pages/sections)

1. **Font loading** -- Load LiberationSans family from `./fonts/`. Map any I/O error to `ExportError::FontLoading`.

2. **Document setup** -- Create `genpdf::Document` with the loaded font family. Set default font size (10pt), title, paper size (A4), margins.

3. **Title page** -- Add analysis name as large heading. Add description if present. Add metadata: input type, status, creation date, processing time, token count.

4. **Executive summary** -- A paragraph summarizing: number of frameworks matched, total findings count, counts by finding type (addressed/partial/gap). Derive these counts by iterating `findings`.

5. **Coverage heatmap** -- Compute coverage per framework: `addressed_count / total_findings_for_framework`. Call `charts::render_coverage_heatmap(&framework_coverages)`. Decode the PNG bytes with the `image` crate, convert to `genpdf::Image`, and push it into the document. If chart rendering fails, log warning and skip (do not fail the entire export).

6. **Per-framework sections** -- For each framework in `frameworks`:
   - Add framework name as section heading
   - Filter findings for this framework
   - Generate radar chart with category labels (finding types) and normalized values, embed as image
   - Add findings table with columns: Priority, Concept Code, Concept Name, Finding Type, Confidence, Evidence (truncated)

7. **Priority breakdown** -- Count findings by priority level. Call `charts::render_priority_chart(...)`. Embed as image.

8. **Appendix** -- If `analysis.extracted_text` is present, include the first 2000 characters as monospace text, with a note that it is truncated.

9. **Render to bytes** -- Use `document.render(&mut writer)` where `writer` is a `Vec<u8>` wrapped in a `std::io::Cursor` or similar. Return the bytes.

#### Chart embedding pattern

```rust
// Pattern for embedding chart PNGs in genpdf
let png_bytes = charts::render_coverage_heatmap(&data)
    .map_err(|e| ExportError::ChartRendering(e.to_string()))?;
let img = image::load_from_memory(&png_bytes)
    .map_err(|e| ExportError::ChartRendering(e.to_string()))?;
let element = genpdf::elements::Image::from_dynamic_image(img)
    .map_err(|e| ExportError::PdfGeneration(e.to_string()))?;
doc.push(element);
```

Note: Check `genpdf` 0.2 API for the exact image embedding method. The crate's `images` feature enables `Image::from_dynamic_image` or similar. If the API differs, adapt accordingly.

#### Findings table rendering

Use `genpdf::elements::TableLayout` (or `LinearLayout` with manual formatting if TableLayout is not available in genpdf 0.2). Each row contains: priority (P1-P4), concept code, concept name (truncated to ~40 chars), finding type, confidence score as percentage, evidence text (first 100 chars).

## Implementation Notes

- **Font loading:** Constructs `FontFamily` manually from 3 font files (Regular, Bold, Italic), reusing Bold as BoldItalic fallback since `genpdf::fonts::from_files` requires all 4 variants
- **Image embedding:** Uses `Image::from_reader(Cursor)` instead of `from_dynamic_image` to avoid image crate version mismatch (genpdf 0.2 depends on image 0.23, project uses 0.25)
- **Chart scaling:** Charts rendered at 800x600px are scaled 0.5x via `CHART_SCALE` constant to fit A4 page margins
- **Chart fallback:** All chart embedding sites show "[Chart could not be rendered]" on failure instead of silent gaps
- **Truncation:** Uses `chars().count()` consistently for both truncation and truncation notice
- **Tests:** 5 tests covering PDF generation (valid bytes, contains title, empty findings), plus truncate utility
- Findings table rendered as formatted paragraphs (one per finding) rather than `TableLayout` for simplicity
